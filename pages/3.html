The forthcoming version: Coq 8.5

Coq 8.5 is available for beta testing. If you wish to use a stable version, see Coq 8.4. The current beta version features:

  • asynchronous edition of documents under CoqIDE to keep working on a proof while Coq checks the other proofs in the background (by Enrico Tassi);
  • universe polymorphism making it possible to reuse the same definitions at various universe levels (by Matthieu Sozeau);
  • primitive projections improving space and time efficiency of records, and adding eta-conversion for records (by Matthieu Sozeau);
  • a new tactic engine allowing dependent subgoals, fully backtracking tactics, as well as tactics which can consider multiple goals together (by Arnaud Spiwack);
  • a new reduction procedure called native_compute to evaluate terms using the OCaml native compiler, for proofs with large computational steps (by Maxime Dénès).

The main other changes are:

  • a new compilation chain that produces .vio files quickly by skipping the checking of proofs;
  • a new construct $(..)$ to call tactics from term definitions;
  • a more restrictive guard condition to recover compatibility with axioms like propositional extensionality and (some consequences of) univalence;
  • a new option -type-in-type to collapse the universe hierarchy (makes the logic inconsistent but useful for exploration);
  • a better-behaved alternative to simpl, called cbn;
  • various improvements to the tactic language, like a new introduction pattern [= x1 ... xn] which applies injection on the fly (inspired by SSReflect);
  • a new construct uconstr:c and type_term c to build untyped terms in Ltac;
  • significant improvements on efficiency in general.

For a full log of changes, see the file CHANGES.

Coq 8.5 is not entirely upward compatible with 8.4

Installation of Coq 8.5

The recommended way for getting Coq depends on the operating system you are running.

For the Windows operating system we provide an installer that bundles Coq and the CoqIDE user interface.

For Unix systems like MacOS X or Linux we recommend to use the OPAM tool to install Coq. Such tool can also be used to install additional Coq libraries and plugins. On Linux systems OPAM can also install CoqIDE. On Mac OS X we provide a package for CoqIDE.
The OPAM tool assists you in the compilation of Coq by providing a clean build environment and resolving the dependencies of Coq, like the OCaml compiler. See the page dedicated to installing Coq via OPAM (coming soon).

Previous and development versions of Coq

The current stable releases of the Coq system is version 8.4 and the previous one was version 8.3.

The development version of Coq is browsable and downloadable from our Git repository using command: git clone git://scm.gforge.inria.fr/coq/coq.git. The daily level of stability of this version (and of the bug-fix branches) can be observed at the coq-bench site or by subscribing to the coq-bench mailing list.

Most of the previous version of Coq are available here.

Extensions of Coq

When compiled on an architecture for which OCaml supports dynamic linking, Coq supports dynamic loading of extensions. The following extensions are known to work as plugins:

  • The small scale reflection package contains the robust and compact tactic language. It is used to write the mathamtical components library on top of which the formal proof of the 4 color theorem and the odd order theorem stand. (*)
  • The rewriting modulo associativity and commutativity plugin (AAC_tactics). (*)
  • The classical extraction module provides extraction of programs from classical proofs in Prop.
  • The descente infinie plugin for delayed inference of the induction hypothesis needed in well-founded induction. (*)

(*) above means that the plugin is submitted to the user contributions and planned to be kept compatible with the new releases of Coq to come.
